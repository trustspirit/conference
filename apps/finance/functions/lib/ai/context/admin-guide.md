### 2. 청구 검토 가이드 (재무 담당자용)

재무 담당자(운영 재정담당, 준비 재정담당)는 제출된 청구서의 1단계 검토를 수행합니다.

#### 검토 체크리스트

1. **영수증 확인**: 모든 지출 항목에 영수증이 첨부되어 있는지 확인
2. **금액 일치 확인**: 영수증 금액과 청구 금액이 일치하는지 확인
3. **예산 코드 확인**: 지출 내용에 적합한 예산 코드가 선택되었는지 확인
   - 숙박/시설 → 5862
   - 교통비 → 5110
   - 식비/보험/행정 → 5400
   - 물품/용품 → 5200
   - 참가비 → 4500
4. **한도 확인**: 식비 1인 1만원 등 한도 준수 여부 확인
5. **10만원 이상**: 세금계산서 또는 카드 영수증이 첨부되었는지 확인
6. **교통비**: 출발지/도착지가 정확하고 경로가 합리적인지 확인
7. **중복 확인**: 동일한 내용의 중복 청구가 아닌지 확인
8. **날짜 확인**: 지출 날짜가 올바른지 확인

#### 검토 처리 방법

- **검토 완료**: 모든 항목이 적정하면 '검토 완료' 버튼을 클릭합니다. → 상태가 검토완료로 변경되며, 결재권자의 승인 단계로 넘어갑니다.
- **반려**: 문제가 있으면 '반려' 버튼을 클릭하고 **반려 사유를 상세히 입력**합니다. → 청구자에게 이메일 알림이 발송됩니다.
- 처리 후 **다음 미처리 청구로 자동 이동**합니다 (남은 건수 뱃지가 표시되며, 슬라이드 애니메이션으로 전환).

#### 청구 관리 목록 기능

- **컬럼 정렬**: 날짜, 수취인, 금액, 상태별로 오름차순/내림차순 정렬이 가능합니다.
- **비고(Remarks) 컬럼**: 각 청구의 추가 정보가 표시됩니다:
  - 재제출된 청구의 경우 **재제출 뱃지**와 원본 청구 링크
  - 검토자/승인자 이름 (마우스 오버 시 툴팁)
  - 반려 사유 (마우스 오버 시 툴팁)
  - 금액이 기준 초과 시 **'상위 결재권자 필요'** 경고 뱃지

#### 업체 신청서 검토 시 유의사항

- 업체 신청서는 수취인이 외부 업체이므로, **통장 사본이 업체의 것**인지 확인해야 합니다 (신청자 본인이 아닌 업체 통장 사본).
- 업체 신청서에 첨부된 통장 사본은 신청서에 별도로 업로드된 것이며, 신청자의 프로필 통장 사본과 다릅니다.
- 업체 신청서는 위원회가 **준비위원회**로 고정되어 있습니다.

#### 운영 재정담당과 준비 재정담당의 차이

| 기능                  | 운영 재정담당 | 준비 재정담당 |

| 준비위 청구 검토      |       X       |       O       |
| 강제 반려 (승인 후)   |       X       |       O       |
| 정산 처리             |       X       |       O       |

| 사용자 목록/역할 변경 |       X       |       O       |
| 대시보드 접근         |       X       |       O       |

### 3. 청구 승인 가이드 (결재권자용)

결재권자는 재무 담당자가 검토를 완료한 청구서를 최종 승인합니다.

#### 승인 절차

1. 관리자 메뉴 → '청구 관리' → 검토완료 상태의 청구를 선택합니다.
2. 지출 내용, 영수증, 금액 등을 최종 확인합니다.
3. **전자 서명을 입력**하고 '승인' 버튼을 클릭합니다.
4. 또는 '반려' 버튼을 클릭하여 반려할 수 있습니다 (반려 사유 입력 필수).
5. 처리 후 **다음 미처리 청구로 자동 이동**합니다.

#### 승인 전 체크리스트

1. 지출 항목이 적정한지
2. 예산이 충분한지
3. 통장 사본의 이름이 수취인과 일치하는지

#### 승인 권한 범위

| 역할          | 승인 가능 위원회 |  금액 제한  | 비고          |
| ------------- | :--------------: | :---------: | ------------- |
| 운영 결재권자 |     운영위만     | 60만원 이하 | 기본 결재권자 |
| 준비 결재권자 |     준비위만     | 60만원 이하 | 기본 결재권자 |
| 운영 위원장   |     운영위만     |  제한 없음  | 상위 결재권자 |
| 준비 위원장   |     준비위만     |  제한 없음  | 상위 결재권자 |
| 대회장        |   모든 위원회    |  제한 없음  | 최고 결재권자 |
| 시스템 관리자 |   모든 위원회    |  제한 없음  | 시스템 관리자 |

#### 특수 규칙

- **본인 청구 승인 불가**: 자기가 제출한 청구서는 승인할 수 없습니다.
- **위원장 청구**: 운영 위원장 또는 준비 위원장이 제출한 청구는 **대회장 또는 시스템 관리자만** 승인할 수 있습니다.
- **금액 초과 청구**: 프로젝트 설정의 기준 금액(기본 60만원)을 초과하는 청구는 일반 결재권자(운영/준비 결재권자)가 승인할 수 없으며, 상위 결재권자가 승인해야 합니다.

### 4. 대시보드 사용 가이드

대시보드는 프로젝트의 예산 현황을 시각적으로 보여줍니다.

**접근 가능 역할**: 준비 재정담당, 운영 위원장, 준비 위원장, 대회장, 시스템 관리자

#### 차트 종류

| 차트                  | 설명                                                       |
| --------------------- | ---------------------------------------------------------- |
| 예산 링 게이지        | 총 예산 대비 현재 사용률을 원형으로 표시                   |
| 예산 코드별 막대 차트 | 5862, 5110, 5400, 5200, 4500 각 코드별 예산 사용 현황 비교 |
| 위원회별 막대 차트    | 운영위/준비위별 예산 사용 비교                             |
| 월별 트렌드 차트      | 일별/월별 지출 추세를 라인 차트로 표시                     |

#### 예산 집계 기준

- **검토완료, 승인, 정산완료** 상태의 청구 금액이 예산 사용으로 집계됩니다.
- 대기중, 반려, 강제반려, 취소된 청구는 예산에 포함되지 않습니다.

#### 예산 설정 (시스템 관리자, 준비 재정담당)

대시보드에서 다음 항목을 직접 설정할 수 있습니다:

- **총 예산**: 프로젝트 전체 예산
- **예산 코드별 예산**: 각 코드별 배정 예산
- **경고 임계값(%)**: 예산 소진률이 이 비율(기본 85%)을 초과하면 경고 배너가 표시됩니다.

### 5. 정산 처리 가이드 (준비 재정담당, 시스템 관리자)

정산은 승인된 청구를 묶어 **정산 보고서**를 생성하는 과정입니다.

#### 정산 절차 (3단계 워크플로우)

**1단계 - 선택 (Select):**
1. **정산 메뉴 → '새 정산'** 클릭
2. **승인된 청구 선택**: 정산할 청구를 체크박스로 선택합니다.
   - **Shift+클릭**으로 여러 건을 범위 선택할 수 있습니다.
3. **자동 그룹핑**: 선택된 청구가 **수취인 + 은행계좌 + 위원회 + 세션이 같은 건끼리 자동 그룹핑**됩니다. 동일인이 여러 건을 청구한 경우 하나의 정산 보고서로 합산됩니다.
   - **업체 신청서는 별도 그룹핑**: 업체 신청서(isVendorRequest)는 일반 신청서와 섞이지 않고 업체끼리만 별도로 그룹핑됩니다.

**2단계 - 검토 (Review):**
4. 선택된 청구를 **건별로 하나씩 순서대로 검토**합니다.
5. 각 청구에 대해 **정산 검토 체크리스트**를 확인합니다:
   - 영수증이 지출 항목과 일치하는지
   - 결재자 서명이 있는지
   - 통장 사본의 이름이 수취인과 일치하는지
6. 검토 중 문제가 발견된 청구는 **개별적으로 강제 반려**할 수 있습니다 (나머지 청구의 정산은 그대로 진행).

**3단계 - 확인 (Summary):**
7. 최종 정산 대상 목록과 합계를 확인합니다.
8. **정산 확정**: 정산을 확정하면:
   - 정산 보고서(Settlement)가 생성됩니다.
   - 선택된 청구서들의 상태가 **정산완료**로 변경됩니다.
   - 각 청구서에 정산 보고서 ID가 기록됩니다.

> **주의**: 한 번의 정산에서 처리할 수 있는 청구 건수에는 시스템 제한이 있습니다(약 500건 이내). 이를 초과하면 정산을 나눠서 진행해야 합니다.

#### 정산 보고서 내용

각 정산 보고서에는 다음이 포함됩니다:

- **문서 번호**: 프로젝트 설정의 문서 번호
- **수취인 정보**: 이름, 전화번호, 은행명, 계좌번호
- **통장 사본**: 수취인의 통장 사본 이미지 (업체 신청서의 경우 신청서에 첨부된 업체 통장 사본 사용)
- **지출 항목 목록**: 합산된 모든 지출 항목 (설명, 예산 코드, 금액)
- **합계 금액**: 총 정산 금액
- **영수증**: 모든 영수증 이미지
- **경로 지도**: 교통비 항목의 경로 지도 이미지 (자가용의 경우 거리×단가 계산 내역 포함)
- **결재자 서명**: 승인자의 전자 서명
- **청구자 서명**: 청구자의 전자 서명

#### 정산 보고서 PDF 출력

정산 보고서를 **PDF로 출력**할 수 있습니다. PDF에는 위의 모든 내용이 포함되며, 서명과 영수증이 인쇄 가능한 형태로 배치됩니다. 영수증 중 PDF 파일은 첫 페이지가 이미지로 변환되어 포함됩니다.

- PDF 출력 시 **'통장 사본 포함'** 체크박스로 통장 사본 이미지 포함 여부를 선택할 수 있습니다.
- PDF에는 **Area Office Finance Verification** 블록이 포함됩니다 (문서 번호, 서명란, 승인일자, 추가 정보/의견 기입란).

#### 배치 리포트

같은 배치에서 생성된 여러 정산은 **배치 리포트**로 묶어 조회할 수 있습니다. 정산 목록에서 배치 ID가 같은 항목은 한 번에 묶어서 확인 가능합니다.

#### 정산 목록 조회

**정산 목록 접근 가능 역할**: 준비 재정담당, 운영 결재권자, 준비 결재권자, 운영 위원장, 준비 위원장, 대회장, 시스템 관리자

정산 목록에서 모든 정산 보고서를 확인하고 개별 보고서의 상세 내용을 조회할 수 있습니다.

### 6. 영수증 관리 가이드 (준비 재정담당, 시스템 관리자)

- **영수증 목록**: 전체 프로젝트의 모든 영수증을 목록으로 조회할 수 있습니다.
- **개별 다운로드**: 영수증 1개를 선택하면 해당 파일을 직접 다운로드합니다.
- **일괄 다운로드**: 여러 영수증을 선택하면 **ZIP 파일로 일괄 다운로드**할 수 있습니다 (파일명에 위원회 필터와 날짜 포함).

### 7. 사용자 관리 가이드 (준비 재정담당, 시스템 관리자)

#### 사용자 목록 조회

모든 프로젝트 소속 사용자의 목록을 조회할 수 있습니다.

- 사용자 이름에 마우스를 올리면 **은행명, 계좌번호, 통장 사본 이미지**가 툴팁으로 표시됩니다. 재무 담당자가 지급 정보를 빠르게 확인할 때 유용합니다.

#### 역할 변경

사용자의 역할을 변경할 수 있습니다. 선택 가능한 역할:

- 일반 사용자, 운영 재정담당, 운영 결재권자, 준비 재정담당, 준비 결재권자, 운영 위원장, 준비 위원장, 대회장, 시스템 관리자

> **참고**: 최고 시스템 관리자(super_admin)는 UI에서 역할을 변경할 수 없습니다. Firestore에서만 설정 가능한 보호된 계정입니다.

#### 사용자 삭제 (시스템 관리자만 가능)

시스템 관리자만 사용자를 삭제할 수 있습니다. 삭제 시:

- 로그인 계정이 삭제됩니다.
- 사용자 데이터가 삭제됩니다.
- 소속 프로젝트의 멤버 목록에서 제거됩니다.
- **본인은 삭제할 수 없습니다.**
- **최고 시스템 관리자(super_admin)는 삭제할 수 없습니다.**

> **주의**: 준비 재정담당은 사용자 목록 조회와 역할 변경은 가능하지만, 사용자 삭제는 불가합니다.

### 8. 프로젝트 관리 가이드 (시스템 관리자만 가능)

#### 프로젝트 생성

새 컨퍼런스/행사 프로젝트를 생성합니다. 설정 항목:

- 프로젝트 이름, 설명
- 문서 번호 (정산 보고서에 표시)
- 총 예산 및 예산 코드별 예산
- 멤버 목록
- 결재 기준 금액 (기본 60만원)
- 예산 경고 임계값 (%, 기본 85%)
- km당 자차 교통비 단가 (기본 ₩250)

#### 프로젝트 삭제 및 복구

- **소프트 삭제**: 삭제하면 즉시 비활성화되지만 데이터는 유지됩니다.
- **30일 복구 기간**: 삭제 후 30일 이내에는 복구할 수 있습니다.
- **자동 영구 삭제**: 30일이 지나면 매일 실행되는 자동 작업에 의해 영구 삭제됩니다 (관련 청구, 정산, 영수증, 경로지도 파일 모두 삭제).

#### 기본 프로젝트 설정

- 새로 가입하는 사용자가 자동으로 배정되는 **기본 프로젝트**를 설정할 수 있습니다.
- 설정 메뉴의 글로벌 설정에서 변경합니다.

#### 멤버 관리

- 프로젝트에 사용자를 추가하거나 제거합니다.
- 프로젝트에 소속된 사용자만 해당 프로젝트의 청구를 제출하고 조회할 수 있습니다.

### 9. 강제 반려 가이드 (준비 재정담당, 시스템 관리자)

이미 **승인**된 청구서에 문제가 발견된 경우, 강제 반려를 수행할 수 있습니다.

**사용 시나리오:**

- 승인 후 영수증 위조/오류가 발견된 경우
- 중복 승인이 확인된 경우
- 예산 코드 또는 금액에 중대한 오류가 있는 경우

**처리 방법:**

1. 청구 관리에서 승인 상태의 해당 청구를 선택합니다.
2. '강제 반려' 버튼을 클릭합니다.
3. 반려 사유를 상세히 입력합니다.
4. 확인하면 상태가 **강제반려**로 변경됩니다.
5. 청구자에게 이메일 알림이 발송됩니다.
6. 청구자는 해당 건을 수정하여 재제출할 수 있습니다.

> **주의**: 이미 정산완료된 청구는 강제 반려할 수 없습니다.
