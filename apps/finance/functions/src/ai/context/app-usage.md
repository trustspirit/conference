## 앱 사용법 (역할별 기능 안내)

이 문서는 컨퍼런스 경비 관리 앱의 모든 기능을 **사용자 역할별**로 설명합니다.

---

### 역할(Role) 개요

앱에는 9가지 역할이 있으며, 각 역할에 따라 접근 가능한 메뉴와 기능이 다릅니다.

| 역할 | 설명 | 주요 권한 |
|------|------|-----------|
| 일반 사용자 | 경비 청구를 제출하는 일반 참가자 | 청구 제출, 본인 내역 조회, 프로필 관리 |
| 운영 재정담당 | 운영위원회 청구를 검토하는 재무 담당자 | 운영위 청구 검토(승인/반려) |
| 준비 재정담당 | 모든 위원회 청구를 검토하고 정산을 처리하는 재무 담당자 | 모든 위원회 청구 검토, 정산 처리, 영수증 관리, 사용자 목록 조회, 대시보드 접근 |
| 운영 결재권자 | 운영위원회 청구를 최종 승인하는 결재권자 | 운영위 청구 승인 (60만원 이하) |
| 준비 결재권자 | 준비위원회 청구를 최종 승인하는 결재권자 | 준비위 청구 승인 (60만원 이하) |
| 운영 위원장 | 운영위원회의 상위 결재권자 | 운영위 청구 무제한 승인, 대시보드 접근, 정산 보고서 조회 |
| 준비 위원장 | 준비위원회의 상위 결재권자 | 준비위 청구 무제한 승인, 대시보드 접근, 정산 보고서 조회 |
| 대회장 | 전체 조직의 최고 결재권자 | 모든 위원회 무제한 승인, 대시보드 접근, 정산 보고서 조회 |
| 시스템 관리자 | 앱 전체를 관리하는 관리자 | 모든 권한 + 사용자 역할 변경 + 프로젝트 관리 + 사용자 삭제 |

### 1. 모든 사용자 공통 기능

#### 1-1. 최초 로그인 및 계정 설정
- 구글(Google) 계정으로 로그인합니다.
- 최초 로그인 시 **필수 정보 입력 모달**이 표시됩니다:
  - 이름 (필수)
  - 전화번호 (필수)
  - 은행명 선택 (필수)
  - 계좌번호 입력 (필수)
  - 소속 위원회 선택: 운영위원회 또는 준비위원회 (필수)
- 이어서 **개인정보 활용 동의** 화면이 표시됩니다. 동의하지 않으면 서비스를 이용할 수 없습니다 (로그아웃만 가능).
- 신규 가입 시 **기본 프로젝트**가 설정되어 있으면 해당 프로젝트에 자동으로 배정됩니다.

#### 1-2. 프로필 설정 (프로필 페이지)
프로필 페이지에서 다음 정보를 관리합니다:

- **이름**: 정산 보고서에 표시되는 이름
- **전화번호**: 연락처 (010-XXXX-XXXX 형식으로 자동 포맷)
- **은행 정보**: 은행명 선택 + 계좌번호 입력 (은행별로 계좌번호 형식과 자릿수가 다르며, 자동으로 형식에 맞게 포맷됩니다)
- **통장 사본**: 이미지 파일 업로드 (PNG, JPG, PDF / 최대 800KB). **통장 사본이 등록되지 않으면 청구서를 제출할 수 없습니다.**
- **전자 서명**: 서명 패드에서 직접 서명을 그려 등록합니다. 정산 보고서에 포함됩니다.
- **기본 위원회**: 운영위원회 또는 준비위원회 중 선택
- **언어 설정**: 한국어 / English 선택 가능 (브라우저 언어 기반 자동 감지)

> **지원 은행**: KB국민, 신한, 우리, 하나, IBK기업, NH농협, 카카오뱅크, 토스뱅크, SC제일, 대구, 부산, 광주, 전북, 제주, 수협, 새마을금고, 신협, 우체국, KDB산업, 경남, 산림조합 (21개)

#### 1-3. 프로젝트 선택
- 상단 헤더의 **프로젝트 선택기(드롭다운)**를 통해 소속 프로젝트를 변경할 수 있습니다.
- 프로젝트가 1개인 경우 자동으로 해당 프로젝트가 선택됩니다.
- 모든 청구와 정산은 **현재 선택된 프로젝트** 기준으로 처리됩니다.
- 프로젝트에 소속되어 있어야 해당 프로젝트를 선택할 수 있습니다.

#### 1-4. AI 챗봇
- 화면 우측 하단의 **채팅 버튼**을 클릭하면 AI 챗봇 패널이 열립니다.
- 앱 사용법, 경비 정책, 예산 관련 질문에 답변합니다.
- 앱 범위 밖의 질문에는 답변하지 않습니다.
- 한 세션에 최대 20개 메시지, 메시지당 최대 2,000자까지 입력 가능합니다.

### 2. 일반 사용자 기능

#### 2-1. 경비 청구 제출 (신규 청구)
메뉴에서 **'신규 청구'**를 클릭하여 새 청구서를 작성합니다.

**입력 항목:**
- **수취인 이름** (필수): 입금받을 사람의 이름
- **전화번호** (필수): 010-XXXX-XXXX 형식
- **은행명** (필수): 지원 은행 중 선택
- **계좌번호** (필수): 은행별 형식에 맞게 입력
- **날짜** (필수): 지출이 발생한 날짜
- **소속 위원회** (필수): 운영위원회 또는 준비위원회
- **세션**: 소속 세션 선택 (한국, 福島, 東京, 長崎, Монгол, Guam, English, Executive 등)
- **비고**: 추가 설명 (선택사항)
- **지출 항목** (최소 1개, **최대 10개**):
  - 항목 설명 (예산 코드 세부 항목에서 선택)
  - 예산 코드 선택 (필수)
  - 금액 입력 (필수, 0원 초과)
  - 교통비인 경우: 출발지/도착지 입력 → 카카오 API로 거리 자동 계산 + 경로 지도 캡처
- **영수증 첨부** (필수, 최소 1개): PNG, JPG, PDF 파일 (최대 **750KB**/파일)

**예산 코드 종류:**
| 코드 | 분류 | 세부 항목 예시 |
|------|------|---------------|
| 5862 | 숙박/시설 | 숙박, 시설 대여, 장비 대여 |
| 5110 | 교통비 | 참가자 교통, 위원 교통, 기타 교통, 회의 교통/숙박 |
| 5400 | 식비/보험/행정 | 식비, 보험료, 홍보비, 통신비, 다과, 회의 식비, 기타 준비비, 기타 행정비 |
| 5200 | 물품/용품 | 활동 재료, 행사 재료, 문구, 티셔츠, 포스터, 팜플렛 |
| 4500 | 참가비 | 참가비 |

**교통비 상세 입력:**
- 이동수단: 자가용 또는 대중교통
- 이동유형: 왕복 또는 편도
- 출발지/도착지: 주소 검색(카카오 지도 API 기반)으로 입력
- 거리 및 경로: 카카오 모빌리티 API를 통해 실제 도로 기준으로 자동 계산
- **자가용 환급금 자동 계산**: 거리(km) × km당 단가(기본 ₩150) × 이동 배수(왕복=2, 편도=1). 금액이 자동으로 입력되며 직접 수정할 수 없습니다.
  - 예: 서울→수원 30km 왕복 = 30 × ₩150 × 2 = ₩9,000
- **대중교통**: 실비를 직접 입력합니다.
- 경로 지도: 자동으로 캡처되어 청구서에 첨부됨

**중복 청구 감지:**
- 제출 시 같은 총액의 대기중 또는 승인된 기존 청구가 있으면 **"중복 청구일 수 있습니다"** 확인 대화 상자가 표시됩니다.
- 중복이 아니라면 확인을 눌러 제출할 수 있습니다.

**자동 저장(Draft):**
- 작성 중인 청구서는 **자동 저장**됩니다 (입력을 멈추고 0.5초 후 자동 저장).
- 페이지를 벗어나도 다시 돌아오면 이전 내용이 복원됩니다.
- 청구서 제출이 완료되면 임시 저장 데이터는 삭제됩니다.
- 작성 중 페이지를 떠나려 하면 **"저장되지 않은 변경사항이 있습니다"** 경고가 표시됩니다.

**제출 전 검증 체크리스트:**
제출 버튼을 누르면 다음 항목을 확인하는 체크리스트가 표시됩니다:
1. 영수증이 지출 항목과 일치하는지
2. 금액이 정확한지
3. 예산 코드가 올바른지
4. 통장 사본의 이름이 수취인과 일치하는지
5. 통장 사본이 올바른지

모든 항목을 체크해야 최종 제출이 가능합니다.

#### 2-2. 내 청구 내역 확인 (내 청구)
메뉴에서 **'내 청구'**를 클릭하면 본인이 제출한 모든 청구 내역을 확인할 수 있습니다.

- **상태별 필터링**: 전체, 대기중, 검토완료, 승인, 정산완료, 반려 등으로 필터링
- **무한 스크롤**: 한 번에 20건씩 로딩되며, 스크롤하면 추가 항목이 자동 로딩됩니다.
- 각 청구를 클릭하면 **상세 페이지**로 이동하여 상태 진행 현황, 검토/승인 이력, 반려 사유 등을 확인할 수 있습니다.

#### 2-3. 청구 상태 흐름
청구서는 다음과 같은 상태 흐름을 따릅니다:

| 상태 | 의미 | 다음 가능한 상태 |
|------|------|------------------|
| 대기중 | 제출 후 재무 담당자의 검토를 기다리는 상태 | → 검토완료 또는 반려 (재무 담당자가 처리) / → 취소 (본인이 직접) |
| 검토완료 | 재무 담당자가 영수증과 내용을 확인 완료한 상태 | → 승인 또는 반려 (결재권자가 처리) |
| 승인 | 결재권자가 최종 승인한 상태 | → 정산완료 (재무 담당자가 정산 처리) / → 강제반려 (재무 담당자 또는 관리자) |
| 정산완료 | 정산 보고서에 포함되어 처리 완료된 상태 | (최종 상태) |
| 반려 | 검토 또는 승인 단계에서 반려된 상태 | 재제출 가능 |
| 강제반려 | 승인 후 재정담당자가 강제로 반려한 상태 | 재제출 가능 |
| 취소 | 본인이 직접 취소한 상태 | (최종 상태) |

#### 2-4. 청구서 재제출
반려 또는 강제반려된 청구서는 **재제출**할 수 있습니다.

1. '내 청구' 목록에서 반려된 청구를 선택합니다.
2. 상세 페이지에서 **'재제출'** 버튼을 클릭합니다.
3. 이전 내용이 자동으로 채워진 청구서 양식이 표시됩니다.
4. 반려 사유를 참고하여 수정한 후 다시 제출합니다.
5. **원본과 달라진 점이 없으면 재제출할 수 없습니다** — 최소 한 곳 이상 수정해야 합니다.

> 재제출된 청구서는 새로운 청구로 생성되며, 원본 청구서와 자동으로 연결됩니다.

#### 2-5. 청구 취소
**대기중** 상태의 청구서만 본인이 직접 취소할 수 있습니다. 검토완료 이후 상태의 청구는 직접 취소할 수 없습니다.

### 3. 재무 담당자 기능 (운영 재정담당 / 준비 재정담당)

재무 담당자는 제출된 청구서를 **검토(review)**하는 역할입니다.

- **운영 재정담당**: 운영위원회 청구만 검토 가능
- **준비 재정담당**: 모든 위원회(운영위 + 준비위) 청구를 검토 가능하며, 추가적으로 정산, 영수증, 사용자 관리, 대시보드에 접근 가능

#### 3-1. 청구 검토 (청구 관리)
관리자 메뉴에서 **'청구 관리'**를 클릭하면 검토 대상 청구 목록이 표시됩니다.

**검토 절차:**
1. 대기중 상태의 청구를 선택합니다.
2. 영수증, 금액, 예산 코드, 항목 설명 등을 확인합니다.
3. **'검토 완료'** 버튼을 클릭하여 검토를 승인하거나, **'반려'** 버튼을 클릭하여 반려합니다.
4. 반려 시 반려 사유를 입력해야 합니다.
5. 처리 후 **다음 미처리 청구로 자동 이동**합니다 (남은 건수가 표시됨).

**주의사항:**
- 본인이 제출한 청구서는 본인이 검토할 수 없습니다.
- 검토 완료된 청구는 결재권자의 승인 단계로 넘어갑니다.

#### 3-2. 강제 반려 (준비 재정담당, 시스템 관리자만 가능)
이미 승인된 청구를 **강제 반려**할 수 있습니다. 정산완료된 건은 강제 반려할 수 없습니다.

#### 3-3. 이메일 알림 (자동)
- 새 청구가 제출되면 해당 위원회의 재무 담당자에게 이메일 알림이 발송됩니다.
- 매주 일요일 오전 9시(한국 시간)에 미처리 청구 현황 다이제스트가 발송됩니다.

### 4. 결재권자 기능 (운영 결재권자 / 준비 결재권자)

결재권자는 재무 담당자가 검토를 완료한 청구서를 **최종 승인**하는 역할입니다.

- **운영 결재권자**: 운영위원회 청구 승인 (60만원 이하)
- **준비 결재권자**: 준비위원회 청구 승인 (60만원 이하)

#### 4-1. 청구 승인 (청구 관리)
1. 관리자 메뉴에서 **'청구 관리'**를 클릭합니다.
2. 검토완료 상태의 청구를 선택합니다.
3. 내용을 확인한 후 **전자 서명**을 입력하고 **'승인'** 버튼을 클릭합니다.
4. 또는 **'반려'** 버튼을 클릭하여 반려할 수 있습니다.
5. 처리 후 **다음 미처리 청구로 자동 이동**합니다.

**승인 체크리스트:**
승인 전에 다음 항목을 확인합니다:
1. 지출 항목이 적정한지
2. 예산이 충분한지
3. 통장 사본의 이름이 수취인과 일치하는지

**금액 제한:**
- 운영/준비 결재권자는 **60만원 이하**(프로젝트 설정 기준)의 청구만 승인 가능합니다.
- 60만원 초과 청구는 상위 결재권자(운영 위원장, 준비 위원장, 대회장, 시스템 관리자)가 승인해야 합니다.

**주의사항:**
- 본인이 제출한 청구서는 본인이 승인할 수 없습니다.
- 운영 위원장/준비 위원장이 제출한 청구는 대회장 또는 시스템 관리자만 승인 가능합니다.

#### 4-2. 정산 보고서 조회
결재권자는 정산 보고서 목록과 개별 보고서를 **조회(읽기 전용)**할 수 있습니다.

### 5. 상위 결재권자 기능 (운영 위원장 / 준비 위원장 / 대회장)

상위 결재권자는 **금액 제한 없이** 청구를 승인할 수 있습니다.

- **운영 위원장**: 운영위원회 청구 무제한 승인
- **준비 위원장**: 준비위원회 청구 무제한 승인
- **대회장**: 모든 위원회 청구 무제한 승인

#### 5-1. 추가 접근 가능 메뉴
일반 결재권자와 달리 다음 메뉴에도 접근할 수 있습니다:
- **대시보드**: 예산 현황, 예산 코드별 분석, 월별 트렌드 확인
- **정산 보고서 조회**: 정산 목록 및 개별 보고서 조회 (읽기 전용)

### 6. 시스템 관리자 기능

시스템 관리자는 모든 권한을 가지며, 추가적으로 다음 기능을 사용할 수 있습니다.

#### 6-1. 사용자 관리
- 모든 사용자 목록을 조회합니다.
- 사용자의 **역할을 변경**할 수 있습니다.
- 사용자를 **삭제**할 수 있습니다 (계정과 데이터가 완전히 삭제되고, 소속 프로젝트에서도 제거됩니다).
- **본인은 삭제할 수 없습니다.**

> 준비 재정담당도 사용자 목록 조회와 역할 변경이 가능하지만, 사용자 삭제는 시스템 관리자만 가능합니다.

#### 6-2. 프로젝트 관리 (설정 메뉴)
- **프로젝트 생성**: 새 프로젝트(컨퍼런스/행사)를 생성합니다.
- **프로젝트 설정 변경**:
  - 문서 번호: 정산 보고서에 표시되는 문서 번호
  - 총 예산: 프로젝트 전체 예산
  - 예산 코드별 예산: 각 코드(5862, 5110, 5400, 5200, 4500)별 예산 배정
  - 결재 기준 금액: 이 금액 초과 시 상위 결재권자 필요 (기본값 60만원)
  - 예산 경고 임계값: 예산 소진율이 이 비율(%)을 초과하면 경고 표시 (기본값 85%)
  - km당 자차 교통비 단가: 자가용 교통비 자동 계산에 사용 (기본값 ₩150)
- **멤버 관리**: 프로젝트에 소속될 사용자를 추가/제거합니다.
- **프로젝트 삭제**: 소프트 삭제 방식으로, 삭제 후 30일 동안 복구 가능합니다. 30일 후 자동으로 영구 삭제됩니다 (관련 청구, 정산, 파일 모두 삭제).
- **기본 프로젝트 설정**: 새 사용자가 자동으로 배정되는 기본 프로젝트를 설정합니다.

#### 6-3. 대시보드
예산 현황을 시각적으로 확인하는 대시보드입니다.

- **예산 링 게이지**: 총 예산 대비 사용 현황을 원형 차트로 표시
- **예산 코드별 막대 차트**: 각 예산 코드별 사용 현황 비교
- **위원회별 막대 차트**: 운영위/준비위별 예산 사용 비교
- **월별 트렌드 차트**: 일별/월별 지출 추세 라인 차트
- **예산 설정**: 대시보드에서 직접 예산 설정을 변경할 수 있습니다.
- **예산 경고 배너**: 설정한 임계값을 초과하면 상단에 경고 배너가 표시됩니다.

> **예산 집계 기준**: 예산 사용량은 **검토완료, 승인, 정산완료** 상태의 청구 금액 합산입니다. 대기중, 반려, 취소된 청구는 예산에 포함되지 않습니다.

#### 6-4. 영수증 관리
- 전체 영수증 목록을 조회합니다.
- 개별 영수증을 다운로드하거나, **일괄 다운로드(ZIP 파일)**할 수 있습니다.

#### 6-5. 정산 처리 (정산 메뉴)
승인된 청구를 묶어 정산 보고서를 생성합니다.

**정산 절차:**
1. 정산 메뉴에서 **'새 정산'**을 클릭합니다.
2. 승인 상태의 청구 중 정산할 건을 선택합니다.
3. **수취인 + 은행계좌 + 위원회 + 세션이 같은 청구끼리 자동 그룹핑**됩니다.
4. 정산 검토 체크리스트를 확인합니다:
   - 영수증이 지출 항목과 일치하는지
   - 결재자 서명이 있는지
   - 통장 사본의 이름이 수취인과 일치하는지
5. **정산 보고서가 생성**되며, 선택된 청구는 정산완료 상태로 변경됩니다.
6. 정산 보고서에는 다음이 포함됩니다:
   - 수취인 정보 (이름, 전화번호, 은행/계좌)
   - 통장 사본
   - 합산된 지출 항목 목록
   - 모든 영수증
   - 결재자 서명
   - 문서 번호
   - 경로 지도 (교통비 항목)
7. **PDF로 출력**하여 서명 후 보관할 수 있습니다.

### 7. 자주 묻는 질문 (FAQ)

**Q: 청구서를 제출할 수 없습니다.**
A: 프로필 페이지에서 통장 사본이 등록되어 있는지 확인하세요. 통장 사본이 없으면 청구서 제출이 제한됩니다. 또한 전자 서명도 등록되어 있어야 합니다.

**Q: 영수증 파일이 업로드되지 않습니다.**
A: 영수증 파일은 PNG, JPG, PDF 형식만 지원하며, 파일당 최대 **750KB**입니다. 파일 크기를 확인해주세요.

**Q: 통장 사본이 업로드되지 않습니다.**
A: 통장 사본은 PNG, JPG, PDF 형식만 지원하며, 최대 **800KB**입니다.

**Q: 내 청구가 반려되었습니다. 어떻게 해야 하나요?**
A: '내 청구' 목록에서 반려된 청구를 선택한 후 '재제출' 버튼을 클릭하세요. 반려 사유를 확인하고 수정하여 다시 제출할 수 있습니다. 단, 원본과 달라진 점이 최소 한 곳 이상 있어야 재제출이 가능합니다.

**Q: 60만원이 넘는 청구는 어떻게 승인받나요?**
A: 60만원 초과 청구는 일반 결재권자가 아닌 상위 결재권자(운영 위원장, 준비 위원장, 대회장, 시스템 관리자)가 승인해야 합니다. 60만원 기준은 프로젝트 설정에서 변경할 수 있습니다.

**Q: 프로젝트를 변경하고 싶습니다.**
A: 상단 헤더의 프로젝트 선택기(드롭다운)에서 변경할 수 있습니다. 프로젝트에 소속되어 있어야 선택 가능합니다.

**Q: 교통비 거리가 자동으로 계산되나요?**
A: 네. 교통비 항목에서 출발지와 도착지를 주소 검색으로 입력하면 카카오 모빌리티 API를 통해 실제 도로 기반 거리와 경로가 자동 계산됩니다. 왕복/편도와 자가용/대중교통도 선택할 수 있습니다.

**Q: 자가용 교통비는 어떻게 계산되나요?**
A: 거리(km) × km당 단가(기본 ₩150) × 이동 배수(왕복=2, 편도=1)로 자동 계산됩니다. 예: 30km 왕복 = 30 × ₩150 × 2 = ₩9,000. 자가용 선택 시 금액이 자동 입력되며 직접 수정할 수 없습니다.

**Q: 이미 승인된 청구를 취소할 수 있나요?**
A: 본인이 직접 취소할 수 있는 것은 대기중 상태의 청구뿐입니다. 이미 승인된 청구는 준비 재정담당 또는 시스템 관리자에게 강제 반려를 요청해야 합니다.

**Q: 서명은 어디서 등록하나요?**
A: 프로필 페이지 하단의 서명 패드에서 직접 서명을 그려 등록할 수 있습니다. 등록된 서명은 정산 보고서에 포함됩니다.

**Q: 앱에서 지원하는 언어는 무엇인가요?**
A: 한국어와 영어를 지원합니다. 프로필 설정 또는 브라우저 언어 설정에 따라 자동으로 감지됩니다.

**Q: 위원회(운영위/준비위)는 무엇인가요?**
A: 컨퍼런스를 운영하는 두 개의 위원회입니다. 운영위원회는 세션 운영을, 준비위원회는 행사 준비를 담당합니다. 각 위원회별로 예산과 결재 라인이 별도로 운영됩니다.

**Q: 정산 보고서 PDF에는 무엇이 포함되나요?**
A: 수취인 정보, 통장 사본, 지출 항목 목록, 합산 금액, 결재자 서명, 청구자 서명, 영수증 이미지, 교통비의 경우 경로 지도, 문서 번호가 포함됩니다.

**Q: 이메일 알림은 언제 받게 되나요?**
A: 다음 경우에 이메일 알림이 발송됩니다:
- 새 청구가 제출될 때 (재무 담당자에게)
- 검토 완료 시 (해당 위원회 결재권자에게 승인 요청)
- 청구가 승인될 때 (청구 제출자에게)
- 청구가 반려/강제반려될 때 (청구 제출자에게)
- 매주 일요일 오전 9시에 미처리 청구 현황 다이제스트 (각 역할별 담당자에게)

**Q: 예산 초과 시에도 청구할 수 있나요?**
A: 청구 제출 자체는 가능하지만, 검토/승인 단계에서 반려될 수 있습니다. 대시보드에서 현재 예산 사용 현황을 확인한 후 청구하는 것을 권장합니다.

**Q: 청구서 작성 중 페이지를 실수로 닫았습니다.**
A: 작성 내용은 자동 저장되므로, 다시 '신규 청구' 페이지에 들어가면 이전 내용이 복원됩니다.

**Q: 한 번에 여러 항목을 청구할 수 있나요?**
A: 네, 하나의 청구서에 최대 10개의 지출 항목을 추가할 수 있습니다. 각 항목마다 예산 코드와 금액을 별도로 입력합니다.

**Q: 삭제된 프로젝트를 복구할 수 있나요?**
A: 프로젝트 삭제 후 30일 이내에는 복구할 수 있습니다. 30일이 지나면 자동으로 영구 삭제되며 복구가 불가능합니다.
